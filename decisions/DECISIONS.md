# DECISIONS

주요 의사결정 기록. 형식: 날짜 - 결정 사항 (이유).

- 2026-07-20: VS Code는 필수로 사용하지 않는다.
- 2026-07-20: GitHub은 백업과 복구 목적으로 사용한다.
- 2026-07-20: 초기 GitHub push는 GitHub Desktop을 사용한다. (Claude Cowork 샌드박스에서 github.com 네트워크 접근이 차단되어 있어 직접 push가 불가능하기 때문)
- 2026-07-20: Slack 채널은 우선 `#lifeos` 하나만 사용한다.
- 2026-07-20: 모든 작업은 Slack 스레드 단위로 관리한다.
- 2026-07-20: 파괴적 작업(GitHub push, 배포, 파일 대량 삭제, DB 파괴적 변경 등)은 사용자 승인이 필요하다.
- 2026-07-20: 기술 스택(매니저 구현 방식 포함)은 설계 문서 확정 후 결정한다. 확정 전까지는 TBD로 표시한다.
- 2026-07-20: 회사명을 CherryBox로 확정하고, 역할에 체리(Founder)/베리(CTO)/망고(Lead Engineer) 명칭을 부여한다. 매니저 역할의 담당(페르소나)은 아직 미정.
- 2026-07-20: 저장소를 회사 운영 저장소(cherrybox 구조, 현재 경로 `lifeos`)와 제품 저장소(`turn-on`)로 분리한다. 회사 저장소는 협업 모델·역할·워크플로우·거버넌스만 다루고, 제품별 구현은 각 제품 저장소에서 진행한다.
- 2026-07-20: CherryBox의 첫 제품은 Turn On(Slack 메시지 캡처)으로 정한다. Sprint 0 범위는 AI 분류/검색/RAG/대시보드 없이 수신→저장(Capture)만 구현하는 것으로 한정한다.
